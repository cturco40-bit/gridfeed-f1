## Change 1 — Standings form strip + trend arrows

Find `renderST` in `index.html` (around line ~945). It builds a `<table class="tt">`
with columns `Pos | Name | Points`. Add two visual enhancements:

### A. Position change arrow

For each driver, compute delta vs. their position at the previous completed
race. Render a small `▲` (green) or `▼` (red) next to their Pos cell,
followed by the magnitude (e.g. `▲2`, `▼1`, or nothing if unchanged).

The jolpica API already returns `previousPosition` in some responses but is
inconsistent. Safest path: query the `historical_races` or `leaderboard` table
for the driver's standing after round `nextRace.round - 1`, or snapshot the
current position into a new lightweight table. A simpler pragmatic approach:
compute the delta from the **previous race's points total** already shown
elsewhere, or just stub the arrow from the `DSTATS` in-memory table if it
carries a `prev_pos` field. **If no clean data source exists, add a TODO
comment and ship arrows only when the underlying data lands — do not
fabricate arrows.**

### B. Form strip

To the right of the name (or as a new column between Name and Points),
render a 5-dot strip showing the driver's last 5 race finishes:
- P1–P3 → gold fill
- P4–P10 → green fill
- P11+ → grey fill
- DNF/DNS → red fill
- Missing race → empty outline

Data source: for local DSTATS (which has `.results` array) you can use
`d.results.slice(-5)`. For the jolpica-merged list, you may need to pull
`historical_performance` table rows via Supabase. Whichever is cleaner.

**Width:** 5 × 10px dots with 3px gap = 62px column. On mobile (<480px),
hide the form strip entirely via `@media(max-width:480px){ .form-strip
{display:none} }`.

Apply both A and B to the drivers table only. Skip the constructors table
for this pass.

## Change 3 — Session countdown chip in top nav

Find the top nav `.top-nav` (around line ~475). Add a small chip to the
right of the nav buttons that shows the next **session** countdown, not
the next **race**.

### Data source
The `SCH` array in `index.html` has race weekends but not per-session times.
You need to derive session times. Two options:

**Option A (preferred if clean):** read from the `schedule` table in Supabase
if it has `scheduled_at` per session. Check its structure first:
```js
const { data } = await sbq(s => s.from('schedule').select('*')
  .gte('scheduled_at', new Date().toISOString())
  .order('scheduled_at', { ascending: true }).limit(1));
```

**Option B (fallback):** hardcode typical session offsets from `SCH[i].d`:
- FP1: Friday local -2d from race day, ~13:30 local
- FP2: Friday -2d, ~17:00 local
- FP3: Saturday -1d, ~12:30 local
- Q:   Saturday -1d, ~16:00 local
- Sprint (if `sp:1`): Saturday -1d, ~11:00 local
- R:   Sunday = `d`, ~14:00 local

This will be approximate — that's fine for the chip. It's a nice-to-have,
not critical data.

### Rendering
```html
<div class="session-chip" id="sessChip" style="display:none">
  <span class="chip-label">NEXT</span>
  <span class="chip-sess" id="sessChipName">—</span>
  <span class="chip-time" id="sessChipTime">—</span>
</div>
```

CSS:
```css
.session-chip{display:inline-flex;align-items:center;gap:6px;padding:4px 10px;
  background:rgba(232,0,45,.12);border:1px solid rgba(232,0,45,.3);
  border-radius:4px;font-size:10px;font-weight:700;color:var(--accent);
  letter-spacing:.5px;margin-left:auto}
.session-chip .chip-label{color:var(--muted);font-size:9px}
.session-chip .chip-sess{color:#fff;font-family:var(--head)}
.session-chip .chip-time{font-variant-numeric:tabular-nums}
@media(max-width:640px){ .session-chip{display:none !important} }
```

The chip hides on mobile — the top nav is already bottom-nav on mobile, so
this is a desktop/tablet header thing only.

Ticking logic: reuse `startCD` pattern but target a different set of DOM
elements. Don't tear down the existing home hero countdown — run them in
parallel.

### When no session is in the next 7 days
